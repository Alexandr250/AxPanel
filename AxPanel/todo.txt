1. «Умная» консоль в папке программы
Вместо того чтобы просто открывать cmd, можно сделать запуск консоли сразу в рабочей директории конкретной кнопки.
Как это реализовать: Добавь в контекстное меню кнопки пункт «Открыть терминал здесь». В коде это Process.Start с аргументом WorkingDirectory.
Фишка: Если программа — это скрипт (Python, Node.js), можно сразу пробрасывать команду активации окружения.
2. Быстрые заметки (Scrapbook)
Вместо запуска тяжелого блокнота, сделай кнопку, которая открывает один и тот же текстовый файл notes.txt из папки лаунчера.
Функционал: Клик — открыл заметки. Правый клик — создал новый .txt в папке «Входящие» с датой в названии.
3. Инструменты «Системного администратора»
Раз у тебя уже есть кнопки выключения и сна, можно добавить:
Kill All: Кнопка, которая принудительно закрывает все процессы, запущенные через лаунчер (у тебя есть список путей в _globalMonitor, так что это легко реализовать через Process.GetProcesses()).
Flush DNS / Clear Temp: Мелкие батники для очистки системы, которые висят в отдельном скрытом контейнере.
4. Drag-and-Drop «Приемник»
Сделай в лаунчере зону (или контейнер), куда можно бросить файл, и он откроется через специфическую программу.
Пример: Кидаешь любой файл на кнопку «VS Code» — он открывается как проект. Кидаешь картинку на кнопку «TinyPNG» (или скрипт сжатия) — она оптимизируется.
5. Режим «Фокус»
Кнопка, которая сворачивает все окна, кроме выбранной программы из лаунчера. Это делается через WinApi (ShowWindow с параметром SW_MINIMIZE).
6. Копирование пути (Copy Path)
Иногда нужно не запустить программу, а быстро получить путь к её .exe или её папке, чтобы вставить куда-то. Зажми Shift + клик по кнопке — путь в буфере обмена.
